# Smart Clinic — User Stories

This document defines user stories for the Smart Clinic Management System, categorized by user roles. Each story outlines what the user wants to achieve and why, ensuring the system design aligns with real-world needs.

---

## 1. Admin User Stories

1. **Admin Login**  
   As an admin, I want to log into the system securely so that I can access the management dashboard.

2. **Manage Doctor Accounts**  
   As an admin, I want to add, edit, or remove doctor accounts so that I can keep the staff list updated.

3. **Manage Patient Accounts**  
   As an admin, I want to view and deactivate patient profiles when necessary for compliance or security purposes.

4. **View System Activity**  
   As an admin, I want to monitor overall system usage and appointment statistics to ensure smooth operations.

---

## 2. Doctor User Stories

1. **Doctor Login**  
   As a doctor, I want to securely log into the system so that I can view my schedule and appointments.

2. **Manage Availability**  
   As a doctor, I want to set my available time slots so that patients can book appointments accordingly.

3. **View Patient Details**  
   As a doctor, I want to view patient records and previous prescriptions before the consultation.

4. **Update Prescriptions**  
   As a doctor, I want to upload or modify a patient’s prescription after the appointment.

---

## 3. Patient User Stories

1. **Patient Registration**  
   As a patient, I want to register easily so that I can book appointments with doctors.

2. **Book Appointment**  
   As a patient, I want to select a doctor and book an appointment based on the doctor’s availability.

3. **View Appointment History**  
   As a patient, I want to view my past and upcoming appointments for better tracking.

4. **Access Prescriptions**  
   As a patient, I want to view prescriptions issued by doctors for easy access and follow-up.
   

---

## 4. Summary

This document defines the functional expectations of each user type—Admin, Doctor, and Patient—supporting the main workflows of the Smart Clinic Management System. These user stories form the foundation for system design, feature prioritization, and testing.
